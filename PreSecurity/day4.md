Day 4 – Introduction to Web Security and HTML Injection

- Front End (Client Side) – the part users see in the browser. 
- Back End (Server Side) – the server that processes requests and sends responses.

Main Technologies Used in Websites
- HTML – builds the structure of a webpage. 
- CSS – adds design and styling. 
- JavaScript – adds interactivity and dynamic features.

HTML uses elements or tags to structure a webpage.
- Attributes like class, src, and id 
- The importance of unique IDs in JavaScript and styling

JavaScript makes websites interactive.

Sensitive Data Exposure happens when developers accidentally leave important information visible in the page source code.
Examples:
- login credentials
- hidden links 
- private information inside comments or scripts

I learned that checking page source (CTRL + U) is useful during web security testing.

HTML Injection happens when a website does not properly filter user input.

Attackers can inject HTML code into input fields, and the browser will render it as real HTML.

Key Takeaways
- Never trust user input. 
- Developers should sanitize and validate inputs. 
- Frontend code can expose sensitive information if not handled properly. 
 HTML and JavaScript are important foundations for understanding web security.
